# AppSail Analytics Runtime Deployment (CAT-005/#21)

The backend (FastAPI + scientific stack) deploys to Catalyst AppSail via
`make deploy-backend` → `scripts/catalyst/deploy_backend.sh`.

## Prerequisites (clean checkout)

1. `npm i -g zcatalyst-cli` and `catalyst login`
2. A Catalyst project (EPIC-CAT/#17; claim code KSPH26) — currently gated on
   **RES-CATALYST-PROJECT-001** (#16 `status:needs-resource`)
3. Environment (never committed — ADR-001):

   | Variable | Purpose |
   |---|---|
   | `CATALYST_PROJECT_ID` | target project |
   | `CATALYST_ORG_ID` | org for `catalyst deploy --org` |
   | `APPSAIL_APP_NAME` | optional, default `kavach-analytics` |

## Runtime contract

- AppSail injects `X_ZOHO_CATALYST_LISTEN_PORT`; the start command binds
  uvicorn to it (`app-config.json` is generated by the deploy script).
- `KAVACH_ENV=catalyst` is set in AppSail env config; all Catalyst
  credentials/ids reach the runtime as AppSail environment variables, not
  files. **No secrets in the repository.**
- Data Store access on AppSail uses `zcatalyst-sdk` (added to the staged
  `requirements.txt` only; local dev keeps the SQLite dev fixture).
