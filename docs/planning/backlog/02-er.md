## Summary
Implement `Accused`, `Victim`, `ComplainantDetails` entities/repositories with enforced semantic guards: PersonID is per-case ordering; complainant demographics are complainant-only; victim PII is sensitivity-tagged.

## Problem Statement
These are the highest-risk tables: person PII plus two documented semantic traps. Graph, entity-resolution and anomaly engines consume them; a wrong abstraction here fabricates identities or discriminatory features.

## Why This Matters
ADR-003/ADR-009 become enforceable code contracts here, not just documents.

## Engineering Objective
Typed entities with sensitivity metadata and guard tests.

## Source Data / ER Schema Mapping (OBSERVED)
- Accused: AccusedMasterID (PK), CaseMasterID (FK), AccusedName, AgeYear, GenderID, PersonID (per-case A1/A2вЂ¦)
- Victim: VictimMasterID (PK), CaseMasterID (FK), VictimName, AgeYear, GenderID, VictimPolice (VARCHAR "1"/"0", Q2)
- ComplainantDetails: ComplainantID (PK), CaseMasterID (FK), ComplainantName, AgeYear, OccupationID, ReligionID, CasteID, GenderID

## ER Conformance Requirements
- PROHIBITED: any cross-case join keyed on Accused.PersonID (guard test required)
- PROHIBITED: ReligionID/CasteID/OccupationID appearing in any engine feature manifest (ADR-009 CI test, delivered with each engine but the manifest convention starts here)
- VictimPolice parsed defensively ("1"/"0"/other в†’ flagged), source type preserved

## Catalyst Services
Catalyst Data Store.

## Dependencies
Blocked by: {{GOV-001}}, {{ER-001}} В· Blocks: {{GRAPH-001}}, {{ENT-001}}, {{ANOM-001}}, {{ER-006}}, {{ER-007}}

## Technical Design
Entities carry `sensitivity` class metadata (HIGH); repository exposes analytics-safe projections (e.g., `AccusedAnalyticsView` without name for aggregate paths). Names retrievable only through case-detail scoped queries.

## Security & Privacy
Names excluded from list/aggregate endpoints by construction; logs never contain names; demographics never leave complainant context.

## Edge Cases
Missing AgeYear; gender code variants (m/f/t vs M/F/T вЂ” normalize in DERIVED view only, preserve raw); duplicate PersonID within a case (flag); accused with no name.

## Acceptance Criteria
- [ ] Guard test fails the build if any repository/engine performs cross-case PersonID equality joins (static scan of query builders + runtime assertion)
- [ ] Entities match documented columns exactly (ER-007 fixture)
- [ ] Analytics projections exclude names and complainant demographics
- [ ] Round-trip tests pass incl. VictimPolice quirk values

## Test Plan

## Definition of Done

## Demo Evidence

## Limitations / Non-Goals

## References
