## Problem
Analytics without an intelligence-story UI is a database. The challenge demands interactive dashboards, geospatial drill-down, relational storytelling — and the UI must visually distinguish OBSERVED FACT from AI_DERIVED from HUMAN_CONFIRMED at all times.

## Why it matters
C2-R1/R2 directly; every other requirement's demo surface. The opening screen must answer "what requires attention now?"

## Technical scope
App shell + auth + classification labeling system, Karnataka map with drill-down, overview dashboard, trends/alerts, network graph, MO, identity review, anomaly+risk, evidence drawer, states/accessibility.

## Out of scope
Backend logic; report PDF (SmartBrowz optional P2 under {{EPIC-DEMO}}).

## Information architecture
1 State Intelligence Overview · 2 Geospatial Intelligence · 3 Emerging Trends · 4 Crime Networks · 5 MO Intelligence · 6 Identity Resolution Review · 7 Anomaly Review · 8 Risk Intelligence · 9 Evidence Detail (drawer) · 10 Admin.

## Catalyst services
Web Client Hosting; Authentication (session); API Gateway.

## Deliverables & success criteria
All screens live against real APIs; classification badges everywhere; SYNTHETIC DATA footer; 5-min demo route flows without dead ends.

## Risks
Map performance; graph rendering scale — capped subgraphs.

## Demo impact
Everything judges see.

## Child issues
{{UI-001}}…{{UI-010}}

## Summary
MapLibre-based Karnataka map: district choropleth (case velocity), hotspot cluster layer (KDE heat + cluster markers, pulsing only when an ACTIVE trend alert covers the area), drill: state → district → station → cluster → case list; filters (category, window, mode spatial/spatiotemporal); every cluster popover links to evidence.

## Problem Statement
C2-R2/R3's primary surface. The pulsing red zone must be driven by real ACTIVE alerts ({{TREND-002}}) — a pulse without an alert is prohibited.

Districts with zero data (render neutral, not alarming); missing-coordinate notice (exclusion count from data-quality); cluster spanning district border; mobile/small screens (min supported width documented).

## Acceptance Criteria
- [ ] Drill path state→district→station→cluster→case works with URL-addressable state
- [ ] Pulse appears ONLY where an ACTIVE alert exists (integration fixture test)
- [ ] Cluster popover: case count, category mix, hour profile, evidence link, classification badge
- [ ] Scoped user sees only their scope's detail layers
- [ ] 60fps pan/zoom on synthetic dataset volume

## Test Plan
Component tests for state machine; manual perf pass; scoped-user visual test.

## Definition of Done
Standard DoD.

## Demo Evidence

## Limitations / Non-Goals
